## 2. Загрузка и декодирование

### 2.1. Поддерживаемые форматы

Приложение поддерживает загрузку аудиофайлов следующих форматов:

| Формат | Расширения | Описание |
|--------|------------|----------|
| WAV | .wav | Несжатый PCM (предпочтительный) |
| MP3 | .mp3 | Сжатый с потерями |
| OGG | .ogg, .oga | Контейнер Ogg Vorbis |
| FLAC | .flac | Сжатие без потерь |
| AAC | .aac, .m4a | Advanced Audio Coding |
| AIFF | .aif, .aiff | Формат Apple |

### 2.2. Процесс декодирования

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        ПРОЦЕСС ДЕКОДИРОВАНИЯ                                 │
└─────────────────────────────────────────────────────────────────────────────┘

ШАГ 1: Пробинг метаданных (ffprobe)
────────────────────────────────────
Вызов: ffprobe -v error -print_format json -show_streams -show_format <file>

Извлекаемые данные:
┌─────────────────────────────────────────────────────────────────────────────┐
│ {                                                                           │
│   "format": {                                                               │
│     "duration": "180.5",        // Длительность в секундах                 │
│     "size": "15000000"          // Размер файла в байтах                   │
│   },                                                                        │
│   "streams": [{                                                             │
│     "codec_type": "audio",                                                  │
│     "sample_rate": "44100",     // Частота дискретизации                   │
│     "channels": 2,              // Число каналов                            │
│     "bits_per_sample": 16       // Битовая глубина                         │
│   }]                                                                        │
│ }                                                                           │
└─────────────────────────────────────────────────────────────────────────────┘

ШАГ 2: Декодирование FFmpeg
───────────────────────────
Вызов: ffmpeg -v error -nostdin -y -i <input> -ac 1 -ar <sr> -f s16le -acodec pcm_s16le -

Параметры:
  -ac 1        → Выход моно (смешивание каналов)
  -ar <sr>     → Сохранение исходной частоты
  -f s16le     → Формат: signed 16-bit little-endian
  -            → Вывод в stdout

Данные на выходе:
  stdout → raw PCM байты (int16)

ШАГ 3: Конвертация в float32
────────────────────────────
buf = np.frombuffer(stdout, dtype=np.int16)    # int16 массив
x = buf.astype(np.float32) / 32768.0           # нормировка [-1.0, 1.0]

Результат:
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   x: numpy.ndarray[float32]  — моно PCM сигнал                             │
│   sr: int                    — частота дискретизации (Гц)                  │
│                                                                             │
│   Размерность x: (n_samples,) где n_samples = sr × duration                │
│   Диапазон значений: [-1.0, 1.0]                                           │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

```
Исходный файл: audio.wav
- Длительность: 5.0 секунд
- Частота: 44100 Гц
```

## 3. Параллельная обработка методами

### 3.1. Общая схема OLA-обработки

Для методов FFT, DCT, DWT, FWHT используется единая схема обработки:

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    СХЕМА OLA ОБРАБОТКИ                                       │
└─────────────────────────────────────────────────────────────────────────────┘

Вход: x[n] — исходный сигнал длины N_total

Параметры OLA:
┌─────────────────────────────────────────────────────────────────────────────┐
│ Параметр           Значение        Описание                                │
│ ─────────────────────────────────────────────────────────────────────────── │
│ block_size (N)     2048            Размер блока (степень двойки)           │
│ hop_size (H)       N/2 = 1024      Шаг между блоками (50% overlap)         │
│ window             sqrt-Hann       Окно анализа и синтеза                  │
│ n_frames           (L-N)/H + 1     Число фреймов                           │
└─────────────────────────────────────────────────────────────────────────────┘

Цикл обработки:

для i = 0, 1, 2, ..., n_frames-1:
    
    ┌─────────────────────────────────────────────────────────────────────────┐
    │                                                                         │
    │   1. Извлечение блока:                                                  │
    │      blk = x[i*H : i*H + N]                                            │
    │                                                                         │
    │   2. Применение окна анализа:                                           │
    │      blk_windowed = blk * w                                            │
    │      где w = sqrt(Hann(N))                                             │
    │                                                                         │
    │   3. Прямое преобразование:                                             │
    │      coeffs = TRANSFORM(blk_windowed)                                  │
    │                                                                         │
    │   4. Отбор коэффициентов (опционально):                                 │
    │      if select_mode == "energy":                                       │
    │          сохранить k% энергии сигнала                                  │
    │      elif select_mode == "lowpass":                                    │
    │          сохранить первые k коэффициентов                              │
    │                                                                         │
    │   5. Обратное преобразование:                                           │
    │      rec = INVERSE_TRANSFORM(coeffs)                                   │
    │                                                                         │
    │   6. Применение окна синтеза:                                           │
    │      rec_windowed = rec * w                                            │
    │                                                                         │
    │   7. Накопление:                                                        │
    │      y_accum[i*H : i*H+N] += rec_windowed                              │
    │      w_accum[i*H : i*H+N] += w²                                        │
    │                                                                         │
    └─────────────────────────────────────────────────────────────────────────┘

Нормировка OLA:

    y = y_accum / w_accum

Защита от клиппинга:

    if max(|y|) > 1.0:
        y = y / max(|y|)

Выход: y[n] — восстановленный сигнал
```

### 3.2. Детальные схемы каждого метода

#### 3.2.1. Метод FFT

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         FFT ПАЙПЛАЙН                                         │
└─────────────────────────────────────────────────────────────────────────────┘

     PCM сигнал
         │
         ▼
    ┌─────────┐
    │ Разбиен.│ ──── Разбиение на блоки с 50% перекрытием
    │ OLA     │
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │ Окно    │ ──── Умножение на sqrt-Hann окно
    │ sqrtHann│      w[n] = √(0.5·(1 - cos(2πn/N)))
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │  rFFT   │ ──── Быстрое преобразование Фурье (real)
    └────┬────┘      X[k] = Σ x[n]·e^(-j·2π·k·n/N)
         │           Возвращает N/2+1 комплексных коэффициентов
         ▼
    ┌─────────┐
    │ Отбор   │ ──── Опционально:
    │коэффици.│      • energy: сохранить k% энергии
    └────┬────┘      • lowpass: сохранить k низких частот
         │
         ▼
    ┌─────────┐
    │  irFFT  │ ──── Обратное преобразование Фурье
    └────┬────┘      x[n] = (1/N)·Σ X[k]·e^(j·2π·k·n/N)
         │
         ▼
    ┌─────────┐
    │ Сборка  │ ──── OLA сборка и нормировка
    │  OLA    │
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │  MP3    │ ──── Кодирование в MP3 через FFmpeg
    └─────────┘


Параметры метода:
┌─────────────────────────────────────────────────────────────────────────────┐
│ Параметр           Значение        Влияние                                 │
│ ─────────────────────────────────────────────────────────────────────────── │
│ block_size         2048            Разрешение по частоте = sr/N            │
│ select_mode        "none"          Режим отбора коэффициентов              │
│ keep_energy_ratio  1.0             Доля сохраняемой энергии (0-1)          │
│ sequency_keep_ratio 1.0            Доля низких частот (0-1)                │
│ bitrate            "192k"          Битрейт выходного MP3                   │
└─────────────────────────────────────────────────────────────────────────────┘
```

#### 3.2.2. Метод DCT

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         DCT ПАЙПЛАЙН                                         │
└─────────────────────────────────────────────────────────────────────────────┘

     PCM сигнал
         │
         ▼
    ┌─────────┐
    │ Разбиен.│
    │ OLA     │
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │ Окно    │
    │ sqrtHann│
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │ DCT-II  │ ──── Дискретное косинусное преобразование
    └────┬────┘      C[k] = √(2/N)·Σ x[n]·cos(π·k·(2n+1)/(2N))
         │           Только действительные коэффициенты!
         ▼
    ┌─────────┐
    │ Отбор   │ ──── Аналогично FFT
    │коэффици.│
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │ IDCT-III│ ──── Обратное косинусное преобразование
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │ Сборка  │
    │  OLA    │
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │  MP3    │
    └─────────┘

Особенность DCT:
- Хорошо концентрирует энергию в низкочастотных коэффициентах
- Используется в JPEG, MP3, AAC
- При keep_energy_ratio = 0.9 сохраняется ~10% коэффициентов
```

#### 3.2.4. Метод FWHT

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         FWHT ПАЙПЛАЙН                                        │
└─────────────────────────────────────────────────────────────────────────────┘

     PCM сигнал
         │
         ▼
    ┌─────────┐
    │ Разбиен.│
    │ OLA     │
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │ Окно    │
    │ sqrtHann│
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │  FWHT   │ ──── Быстрое преобразование Уолша-Адамара
    └────┬────┘      Только сложения и вычитания!
         │           Y = Hₙ · X / √N (ортонормированное)
         ▼
    ┌─────────┐
    │ Отбор   │
    │коэффици.│
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │ IFWHT   │ ──── Обратное преобразование
    └────┬────┘      X = Hₙ · Y / √N
         │
         ▼
    ┌─────────┐
    │ Сборка  │
    │  OLA    │
    └────┬────┘
         │
         ▼
    ┌─────────┐
    │  MP3    │
    └─────────┘

Особенность FWHT:
- O(N log N) операций сложения/вычитания
- Нет умножений — вычислительно эффективно
- Базис — прямоугольные волны (функции Уолша)
```

#### 3.2.5. Метод Huffman-like (μ-law)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    HUFFMAN-LIKE ПАЙПЛАЙН (μ-law)                            │
└─────────────────────────────────────────────────────────────────────────────┘

     PCM сигнал x[n]
         │
         ▼
    ┌─────────────┐
    │ μ-law       │ ──── Логарифмическая компрессия
    │ компрессия  │      y = sign(x)·ln(1 + μ·|x|)/ln(1 + μ)
    └──────┬──────┘      μ = 255 (стандарт G.711)
           │
           ▼
    ┌─────────────┐
    │ Квантование │ ──── Равномерное квантование
    │             │      Q = 2^bits уровней (обычно 256)
    └──────┬──────┘      xi = round((y+1)/2 · (Q-1))
           │
           ▼
    ┌─────────────┐
    │ Обратное    │ ──── Восстановление амплитуды
    │ квантование │      x_rec = xi/(Q-1) · 2 - 1
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │ μ-law       │ ──── Экспандирование
    │ экспандир.  │      x = sign(y)·((1+μ)^|y| - 1)/μ
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │    MP3      │
    └─────────────┘

Параметры:
- mu=255: параметр компрессии (стандарт телефонии)
- bits=8: битовая глубина квантования
```

#### 3.2.6. Метод Rosenbrock-like

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    ROSENBROCK-LIKE ПАЙПЛАЙН                                  │
└─────────────────────────────────────────────────────────────────────────────┘

     PCM сигнал x[n]
         │
         ▼
    ┌─────────────┐
    │ Нелинейное  │ ──── Мягкое сглаживание
    │ преобразов. │      y = x / (1 + α·(x - β)²)
    └──────┬──────┘
           │
           │    Характеристика при разных α:
           │    ┌─────────────────────────────────────────────────────────────┐
           │    │                                                             │
           │    │  Выход                                                      │
           │    │    ∧    α=0 (линейная)                                      │
           │    │    │   /                                                     │
           │    │    │  /  α=0.2                                               │
           │    │    │ / ╭────                                                 │
           │    │    │╱ ╭╯                                                     │
           │    │    └──────────────► Вход                                    │
           │    │       -1  β  1                                              │
           │    └─────────────────────────────────────────────────────────────┘
           ▼
    ┌─────────────┐
    │  Нормировка │ ──── Защита от клиппинга
    │    пика     │      if max(|y|) > 1: y = y / max(|y|)
    └──────┬──────┘
           │
           ▼
    ┌─────────────┐
    │    MP3      │
    └─────────────┘

Параметры:
- alpha=0.2: интенсивность сглаживания (0-10)
- beta=1.0: точка перегиба (-5 до 5)
```

## 4. Кодирование в MP3

### 4.1. Процесс кодирования

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        КОДИРОВАНИЕ В MP3                                     │
└─────────────────────────────────────────────────────────────────────────────┘

Вход: y[n] — PCM float32 моно сигнал [-1.0, 1.0]

ШАГ 1: Подготовка буфера
─────────────────────────
    y_clipped = clip(y, -1.0, 1.0)              # Защита от выхода за границы
    y_int = round(y_clipped * 32767).int16      # Конвертация в int16
    buf = y_int.tobytes()                        # Сериализация в байты

ШАГ 2: Кодирование FFmpeg
─────────────────────────
    Вызов: ffmpeg -v error -y \
             -f s16le -ac 1 -ar <sr> -i pipe:0 \
             -vn \
             -q:a <quality> \
             <output.mp3>

    Параметры:
    -f s16le     — входной формат: signed 16-bit little-endian
    -ac 1        — моно
    -ar <sr>     — частота дискретизации
    -i pipe:0    — чтение из stdin
    -vn          — без видео
    -q:a <q>     — качество VBR (0=лучшее, 9=худшее)

    Сопоставление bitrate → quality:
    ┌─────────────────────────────────────────────────────────────────────────┐
    │ Bitrate    →  Quality                                                  │
    │ ─────────────────────────────────────────────────────────────────────── │
    │ 320k       →  0 (лучшее)                                               │
    │ 256k       →  1                                                        │
    │ 192k       →  2                                                        │
    │ 160k       →  4                                                        │
    │ 128k       →  5                                                        │
    └─────────────────────────────────────────────────────────────────────────┘

    stdin ← buf  (передача PCM данных через pipe)

Выход: <output.mp3> — MP3 файл
```

### 4.2. Режимы кодирования

| Режим | Параметр | Особенности |
|-------|----------|-------------|
| CBR | -b:a 192k | Постоянный битрейт, предсказуемый размер |
| VBR | -q:a 2 | Переменный битрейт, лучшее качество при том же размере |


### 5.2. Детальный расчёт метрик

#### Временные метрики

```
SNR (Signal-to-Noise Ratio):
─────────────────────────────
noise = ref - test
P_signal = mean(ref²)
P_noise = mean(noise²)
SNR = 10·log₁₀(P_signal / P_noise) [дБ]

RMSE (Root Mean Square Error):
───────────────────────────────
error = ref - test
RMSE = √(mean(error²))

SI-SDR (Scale-Invariant SDR):
──────────────────────────────
α = dot(ref, test) / dot(ref, ref)
y_target = α · ref
e_noise = test - y_target
SI-SDR = 10·log₁₀(||y_target||² / ||e_noise||²) [дБ]
```

#### Спектральные метрики

```
Для каждого окна (n_fft=1024, hop=512):

R = |rFFT(ref_window)|    — амплитудный спектр референса
T = |rFFT(test_window)|   — амплитудный спектр теста

LSD (Log-Spectral Distance):
─────────────────────────────
S_R = 20·log₁₀(R)
S_T = 20·log₁₀(T)
LSD_frame = √(mean((S_R - S_T)²))
LSD = mean(LSD_frame) по всем окнам

Spectral Convergence:
──────────────────────
SC_frame = ||R - T||₂ / ||R||₂
SC = mean(SC_frame) по всем окнам

Spectral Centroid Δ:
─────────────────────
cent_R = Σ(k·R_k) / ΣR_k × (sr/N)
cent_T = Σ(k·T_k) / ΣT_k × (sr/N)
CentroidΔ = mean(|cent_R - cent_T|) по всем окнам

Cosine Similarity:
──────────────────
CosSim = dot(R, T) / (||R||·||T||)
```

### 5.3. Агрегированный балл

```
ШАГ 1: Min-Max нормировка
─────────────────────────
Для всех 7 методов вычисляем min и max каждой метрики.

Для метрик «ниже-лучше» (LSD, SC, RMSE, CentroidΔ, time):
    norm = (max - value) / (max - min)

Для метрик «выше-лучше» (SNR, SI-SDR, CosSim):
    norm = (value - min) / (max - min)

ШАГ 2: Взвешенная сумма
───────────────────────
score = 0.20·LSD_n + 0.15·SC_n + 0.20·SNR_n + 
        0.15·RMSE_n + 0.15·SI-SDR_n +
        0.05·CentroidΔ_n + 0.05·CosSim_n + 0.05·time_n

Результат:
• score → 1.0: лучший метод
• score → 0.0: худший метод
```

Все методы обработки наследуются от `BaseTransform`:

```python
class BaseTransform(ABC):
    """Абстрактный базовый класс для всех методов трансформации."""
    
    NAME: str           # Имя метода для логирования
    DESCRIPTION: str    # Краткое описание
    FILE_SUFFIX: str    # Суффикс выходного файла
    
    def __init__(self, block_size, bitrate, select_mode, 
                 keep_energy_ratio, sequency_keep_ratio):
        ...
    
    @abstractmethod
    def transform_block(self, block, **params) -> np.ndarray:
        """Применить преобразование к одному блоку."""
        pass
    
    @abstractmethod
    def process(self, wav_path, out_dir, progress_cb) -> Tuple[str, float]:
        """Полный пайплайн обработки файла."""
        pass
```

### 8.4. Создание новой трансформации

Для добавления нового метода обработки:

1. **Создать файл** `processing/transforms/my_transform.py`:

```python
"""MyTransform: Описание метода."""
from __future__ import annotations
import numpy as np
from .base import BaseTransform

class MyTransform(BaseTransform):
    """Описание новой трансформации."""
    
    NAME = "MyTransform"
    DESCRIPTION = "Мой метод обработки"
    FILE_SUFFIX = "my"
    
    def __init__(self, block_size=2048, **kwargs):
        super().__init__(block_size=block_size, **kwargs)
    
    def transform_block(self, block: np.ndarray, **params) -> np.ndarray:
        """Применить преобразование к блоку."""
        # Прямое преобразование
        coeffs = self._forward_transform(block)
        
        # Отбор коэффициентов (если нужен)
        coeffs = self._select_coefficients(coeffs)
        
        # Обратное преобразование
        return self._inverse_transform(coeffs)
    
    def process(self, wav_path, out_dir, progress_cb=None):
        """Полный пайплайн обработки."""
        # Реализация с OLA или без
        ...
```

2. **Зарегистрировать** в `transforms/__init__.py`:


### 8.6. Пример: реализация FFT-трансформации

```python
class FFTTransform(BaseTransform):
    """FFT-трансформация с OLA и отбором коэффициентов."""
    
    NAME = "FFT"
    DESCRIPTION = "Быстрое преобразование Фурье"
    FILE_SUFFIX = "fft"
    
    def transform_block(self, block: np.ndarray, **params) -> np.ndarray:
        """Применить FFT к одному блоку."""
        # Прямое rFFT
        coeffs = np.fft.rfft(block)
        
        # Отбор коэффициентов по режиму
        coeffs = apply_fft_coefficient_selection(
            coeffs,
            self.select_mode,
            self.keep_energy_ratio,
            self.sequency_keep_ratio,
        )
        
        # Обратное irFFT
        return np.fft.irfft(coeffs, n=len(block)).astype(np.float32)
    
    def process(self, wav_path, out_dir, progress_cb=None):
        """Полный пайплайн FFT обработки."""
        t0 = time.perf_counter()
        
        # Загрузка
        x, sr = load_audio_safe(wav_path)
        n = len(x)
        
        # Подготовка OLA
        win = create_ola_window(self.block_size)
        frames, hop, total_len, y_accum, w_accum = prepare_ola_buffers(n, self.block_size)
        
        # Дополнение и обработка блоков
        x_padded = np.pad(x, (0, total_len - n), mode="constant")
        
        for fi in range(frames):
            i0 = fi * hop
            blk = x_padded[i0 : i0 + self.block_size] * win
            rec = self.transform_block(blk) * win
            y_accum[i0 : i0 + self.block_size] += rec
            w_accum[i0 : i0 + self.block_size] += win * win
            
            if progress_cb:
                progress_cb(0.1 + 0.8 * (fi + 1) / frames, f"FFT: блок {fi+1}/{frames}")
        
        # Финализация
        y = finalize_ola(y_accum, w_accum, n)
        out_mp3 = self.get_output_path(wav_path, out_dir)
        
        # Кодирование в MP3
        encode_pcm_to_mp3(y, sr, out_mp3, self.bitrate)
        
        return out_mp3, time.perf_counter() - t0
```
