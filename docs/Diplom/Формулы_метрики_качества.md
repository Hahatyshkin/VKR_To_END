# Формулы метрик качества аудиосигнала

Все формулы соответствуют реализации в файле `AudioAnalyzer/src/processing/metrics.py`.

---

## Общие обозначения

| Обозначение | Описание |
|---|---|
| `x[n]` (или `ref[n]`) | Опорный (оригинальный) сигнал, массив PCM-отсчётов |
| `y[n]` (или `test[n]`, `tst[n]`) | Тестовый (обработанный) сигнал, массив PCM-отсчётов |
| `N` | Длина общей части сигналов: `N = min(len(x), len(y))` |
| `X_m[k]` | Комплексный спектр опорного сигнала в окне `m` |
| `Y_m[k]` | Комплексный спектр тестового сигнала в окне `m` |
| `sr_ref` | Частота дискретизации опорного сигнала (Гц) |
| `sr_test` | Частота дискретизации тестового сигнала (Гц) |
| `n_fft` (или `N_fft`) | Размер окна FFT (по умолчанию 1024 для спектральных метрик) |
| `hop` (или `H`) | Шаг сдвига окна (по умолчанию 512) |
| `w[n]` | Весовая функция окна (оконная функция Ханна) |
| `K` | Число частотных бинов: `K = n_fft / 2 + 1` |
| `M` | Число окон (фреймов) в сигнале |
| `eps` | Малая константа для избежания деления на ноль: `1e-12` |
| `df` | Частотное разрешение: `df = sr / n_fft` (Гц на бин) |

## 1. SNR (Signal-to-Noise Ratio)

**Функция:** `compute_snr_db(reference, test)`
**Единица:** дБ | **Направление:** выше — лучше

**Описание.** Отношение мощности полезного сигнала к мощности шума (разности между опорным и тестовым). Чем выше SNR, тем меньше искажений внесено при обработке.

**Формула:**

```
noise[n] = x[n] - y[n]                        (вектор ошибки, шум)

P_signal = (1/N) * sum_{n=0}^{N-1} ( x[n]^2 ) + eps    (мощность сигнала)

P_noise  = (1/N) * sum_{n=0}^{N-1} ( noise[n]^2 ) + eps (мощность шума)

SNR = 10 * log10( P_signal / P_noise )    [дБ]
```

**Пояснения:**
- `noise[n]` — разница между опорным и обработанным сигналом (ошибка восстановления)
- `P_signal` — средняя мощность опорного сигнала
- `P_noise` — средняя мощность шума/искажений
- `+ eps` — защита от деления на ноль
- Результат в децибелах: положительный SNR означает, что сигнал сильнее шума

## 2. RMSE (Root Mean Square Error)

**Функция:** `compute_rmse(reference, test)`
**Единица:** безразмерная (0..1 для сигналов нормализованных к [-1, 1]) | **Направление:** ниже — лучше

**Описание.** Корень из среднего квадрата отклонения тестового сигнала от опорного. Показывает среднюю величину ошибки во временной области.

**Формула:**

```
e[n] = x[n] - y[n]                          (вектор ошибки)

RMSE = sqrt( (1/N) * sum_{n=0}^{N-1} ( e[n]^2 ) )
```

**Пояснения:**
- `e[n]` — разница между отсчётами опорного и тестового сигналов
- Для PCM-сигналов в диапазоне [-1, 1]: RMSE = 0 означает идеальное совпадение, RMSE = 1 — максимально возможное отклонение
- RMSE чувствителен к крупным выбросам (в отличие от MAE)

## 3. SI-SDR (Scale-Invariant Signal-to-Distortion Ratio)

**Функция:** `compute_si_sdr_db(reference, test)`
**Единица:** дБ | **Направление:** выше — лучше

**Описание.** Масштабно-инвариантное отношение сигнал/искажение. В отличие от обычного SNR, SI-SDR автоматически подбирает оптимальный масштаб (зум), компенсируя глобальные изменения громкости. Инвариантен к линейному усилению/ослаблению сигнала.

**Формула:**

```
alpha   = sum_{n=0}^{N-1} ( x[n] * y[n] ) / ( sum_{n=0}^{N-1} ( x[n]^2 ) + eps )

y_hat[n] = alpha * x[n]                   (проекция тестового сигнала на опорный)

e_noise[n] = y[n] - y_hat[n]              (шумовая компонента)

SI-SDR = 10 * log10( sum(y_hat[n]^2) + eps / sum(e_noise[n]^2) + eps )    [дБ]
```

**Пояснения:**
- `alpha` — коэффициент масштаба, вычисляемый методом наименьших квадратов (проекция `y` на `x`)
- `y_hat[n]` — «целевой» компонент тестового сигнала (часть, совпадающая с опорным)
- `e_noise[n]` — «шумовая» компонента (искажения)
- Масштабная инвариантность: если `y[n] = c * x[n]` при любом `c != 0`, то SI-SDR = +inf

## 4. LSD (Log-Spectral Distance)

**Функция:** `compute_lsd_db(reference, test, sr_ref, sr_test, n_fft=1024, hop=512)`
**Единица:** дБ | **Направление:** ниже — лучше

**Описание.** Логарифмическое спектральное расстояние. Измеряет среднее различие между спектрами опорного и тестового сигналов в логарифмическом масштабе (дБ). Учитывает частотные характеристики искажений, которые не видны во временной области.

**Предобработка:** если `sr_test != sr_ref`, тестовый сигнал приводится к `sr_ref` линейной интерполяцией.

**Формула (для каждого окна m):**

```
x_m[n] = x[m*H : m*H + n_fft]            (фрагмент опорного сигнала)
y_m[n] = y[m*H : m*H + n_fft]            (фрагмент тестового сигнала)

X_m[k] = rFFT( x_m[n] * w[n] )           (одностороннее FFT)
Y_m[k] = rFFT( y_m[n] * w[n] )

S_x[k] = 10 * log10( |X_m[k]|^2 + eps )  (лог-амплитуда спектра опорного)
S_y[k] = 10 * log10( |Y_m[k]|^2 + eps )  (лог-амплитуда спектра тестового)

LSD_m = sqrt( (1/K) * sum_{k=0}^{K-1} ( S_x[k] - S_y[k] )^2 )

LSD = (1/M) * sum_{m=0}^{M-1} LSD_m       [дБ, среднее по всем окнам]
```

**Пояснения:**
- `w[n]` — окно Ханна: `w[n] = 0.5 * (1 - cos(2*pi*n / (n_fft - 1)))`, уменьшает эффекты спектрального утечек
- `rFFT` — одностороннее быстрое преобразование Фурье (только неотрицательные частоты)
- `S_x[k]`, `S_y[k]` — спектральная плотность мощности в дБ
- `K = n_fft / 2 + 1` — число частотных бинов (для n_fft=1024: K=513)
- `M` — число окон, вычисляемое как `max(1, 1 + (N - n_fft) // hop)`
- LSD = 0 означает идентичные спектры

## 5. Spectral Convergence (Спектральная сходимость)

**Функция:** `compute_spectral_convergence(reference, test, sr_ref, sr_test, n_fft=1024, hop=512)`
**Единица:** безразмерная | **Направление:** ниже — лучше

**Описание.** Относительная разница амплитудных спектров. Показывает, насколько амплитудный спектр обработанного сигнала отклоняется от опорного. Чем ниже значение, тем лучше спектральное совпадение.

**Предобработка:** ресемплирование теста к `sr_ref`.

**Формула (для каждого окна m):**

```
X_m[k] = rFFT( x_m[n] * w[n] )
Y_m[k] = rFFT( y_m[n] * w[n] )

numerator   = || |X_m| - |Y_m| ||_2 = sqrt( sum_{k=0}^{K-1} ( |X_m[k]| - |Y_m[k]| )^2 )
denominator = || |X_m| ||_2            = sqrt( sum_{k=0}^{K-1} ( |X_m[k]| )^2 )

SC_m = numerator / (denominator + eps)

SC = (1/M) * sum_{m=0}^{M-1} SC_m      [среднее по окнам]
```

**Пояснения:**
- `|X_m[k]|` — амплитуда (модуль) комплексного спектра в бине `k`
- `|| ... ||_2` — евклидова норма (L2-норма) вектора
- SC = 0 означает полное совпадение амплитудных спектров

## 6. Spectral Centroid Difference (Разница спектральных центроидов)

**Функция:** `compute_spectral_centroid_diff_hz(reference, test, sr_ref, sr_test, n_fft=1024, hop=512)`
**Единица:** Гц | **Направление:** ниже — лучше

**Описание.** Разница «центров тяжести» спектров опорного и тестового сигналов. Спектральный центроид отражает perceived «яркость» звука: низкий центроид (~500-2000 Гц) характерен для басовых звуков, высокий (~5000-15000 Гц) — для ярких, шипящих.

**Предобработка:** ресемплирование теста к `sr_ref`.

**Формула (для каждого окна m):**

```
A_m[k] = | rFFT( x_m[n] * w[n] ) |     (амплитуды спектра опорного)
B_m[k] = | rFFT( y_m[n] * w[n] ) |     (амплитуды спектра тестового)

df = sr_ref / n_fft                       (частотное разрешение, Гц на бин)

centroid_x = ( sum_{k=0}^{K-1} ( k * A_m[k] ) / ( sum_{k=0}^{K-1} A_m[k] + eps ) ) * df    [Гц]
centroid_y = ( sum_{k=0}^{K-1} ( k * B_m[k] ) / ( sum_{k=0}^{K-1} B_m[k] + eps ) ) * df    [Гц]

d_m = | centroid_x - centroid_y |         (абсолютная разница, Гц)

SC_diff = (1/M) * sum_{m=0}^{M-1} d_m    [Гц]
```

**Пояснения:**
- `k` — индекс частотного бина (0, 1, 2, ..., K-1)
- `k * A_m[k]` — «момент» спектра (взвешенная частота)
- Умножение на `df` переводит результат из единиц бинов в герцы
- Сумма `sum(A_m[k])` — полная энергия спектра в окне

## 7. Spectral Cosine Similarity (Косинусная мера сходства спектров)

**Функция:** `compute_spectral_cosine_similarity(reference, test, sr_ref, sr_test, n_fft=1024, hop=512)`
**Единица:** 0..1 | **Направление:** выше — лучше

**Описание.** Косинус угла между векторами амплитудных спектров. Значение 1.0 означает идентичную форму спектра, 0.0 — ортогональные (полностью различные) спектры. Не чувствительна к глобальным изменениям амплитуды.

**Предобработка:** ресемплирование теста к `sr_ref`.

**Формула (для каждого окна m):**

```
A_m[k] = | rFFT( x_m[n] * w[n] ) |     (амплитудный спектр опорного)
B_m[k] = | rFFT( y_m[n] * w[n] ) |     (амплитудный спектр тестового)

numerator   = sum_{k=0}^{K-1} ( A_m[k] * B_m[k] )
denominator = || A_m || * || B_m || + eps
             = sqrt( sum(A_m[k]^2) ) * sqrt( sum(B_m[k]^2) ) + eps

cos_m = numerator / denominator           [0..1]

CosSim = (1/M) * sum_{m=0}^{M-1} cos_m
```

**Пояснения:**
- Формула аналогична косинусному сходству текстов в NLP
- `numerator` — скалярное произведение двух амплитудных векторов
- `denominator` — произведение их L2-норм

## 8. STOI (Short-Time Objective Intelligibility Index)

**Функция:** `compute_stoi_simplified(reference, test, sr_ref, sr_test)` (упрощённая версия: `n_fft=256, hop=128, n_bands=8`)
**Единица:** 0..1 | **Направление:** выше — лучше

**Описание.** Индекс разборчивости речи на основе кратковременных спектральных огибающих. Сравнивает огибающие опорного и тестового сигналов в октавных полосах частот. Является предиктором разборчивости речи (не общего качества).

**Предобработка:** ресемплирование теста к `sr_ref`.

**Формула:**

```
1. Октавные полосы (1/3 октавы):
   f_c[b] = 150 * 2^(b/3),  b = 0, 1, ..., B-1
   где B = n_bands (8 для упрощённой версии)
   Полоса отсекается по условию: f_c < sr_ref / 2

2. Ширина каждой полосы:
   bw[b] = f_c[b] * ( 2^(1/6) - 2^(-1/6) )

3. Для каждого фрейма m (окна) и полосы b:
   mask = { k : freq_bins[k] >= f_c - bw/2 AND freq_bins[k] <= f_c + bw/2 }

   env_ref[m, b] = sqrt( mean( |FFT_ref[m, mask]|^2 ) )     (огибающая опорного)
   env_tst[m, b] = sqrt( mean( |FFT_tst[m, mask]|^2 ) )     (огибающая тестового)

4. Нормировка огибающих по максимальному значению в каждой полосе:
   env_ref = env_ref / ( max_frame(env_ref) + eps )
   env_tst = env_tst / ( max_frame(env_tst) + eps )

5. Корреляция Пирсона для каждой полосы b:
   r_c = sum( (env_ref - mean(env_ref)) * (env_tst - mean(env_tst)) ) /
         sqrt( sum( (env_ref - mean(env_ref))^2 ) * sum( (env_tst - mean(env_tst))^2 ) ) + eps

6. STOI = clip( mean(r_c), 0, 1 )        [ограничение диапазона [0, 1]]
```

**Пояснения:**
- `f_c[b]` — центральная частота b-й октавной полосы
- `bw[b]` — ширина полосы на уровне -3 дБ
- `env_ref`, `env_tst` — матрицы огибающих размером (M x B)
- `r_c` — коэффициент корреляции Пирсона между огибающими в полосе b
- `clip` — ограничение значения: отрицательная корреляция даёт 0, а не среднее
- Значения: > 0.75 — хорошая разборчивость, > 0.9 — отличная

## 9. PESQ (Perceptual Evaluation of Speech Quality) — приближённый

**Функция:** `compute_pesq_approx(reference, test, sr_ref, sr_test)`
**Единица:** -0.5..4.5 | **Направление:** выше — лучше

**Описание.** Приближённая оценка перцептивного качества речи по стандарту ITU-T P.862. Учитывает психоакустические особенности слуха через A-взвешивание частотных компонент. Является приближённой реализацией (полная требует библиотеки ITU-T).

**Предобработка:** ресемплирование обоих сигналов к 16 кГц.

**Формула:**

```
1. A-взвешивание (упрощённая кривая частотной коррекции):
   W(f) = (12194^2 * f^4) / ( (f^2 + 20.6^2) * sqrt( (f^2 + 107.7^2) * (f^2 + 737.9^2) ) * (f^2 + 12194^2) )
   W(f) = W(f) / max( W(f) )             (нормировка к [0, 1])
   W(0) = 0                               (DC-компонент исключается)

2. Для каждого фрейма m (n_fft=512, hop=256):
   X_m[k] = rFFT( x_m[n] * w[n] )
   Y_m[k] = rFFT( y_m[n] * w[n] )

   diff[k] = | log10( (|X_m[k]| + eps) / (|Y_m[k]| + eps) + eps ) |   (взвешенная лог-разность)

   D_m = mean( (diff * W)^2 )             (средний квадрат взвешенной разности)

3. D = sqrt( mean(D_m) )                   (среднеквадратичное искажение)

4. PESQ = max( -0.5, min( 4.5, 4.5 - 3.0 * D ) )    [эмпирический маппинг]
```

**Пояснения:**
- `W(f)` — частотная коррекция, имитирующая кривую слышимости человека (A-взвешивание)
- `12194`, `20.6`, `107.7`, `737.9` — константы стандарта IEC 61672-1 для A-взвешивания
- `diff[k]` — логарифмическая разница амплитуд (в децибелах)
- Коэффициент `k = 3.0` подобран эмпирически для приближённого маппинга на шкалу PESQ

## 10. MOS (Mean Opinion Score) на основе PESQ

**Функция:** `compute_pesq_mos(reference, test, sr_ref, sr_test)`
**Единица:** 1..5 | **Направление:** выше — лучше

**Описание.** Средняя оценочная оценка качества, полученная преобразованием PESQ по формуле ITU-T P.862.1. Шкала MOS соответствует субъективной оценке слушателей.

**Формула:**

```
MOS = max( 1.0, min( 5.0, 1 + 3.87 / ( 1 + exp( -1.3669 * (PESQ - 0.7197) ) ) ) )
```

**Пояснения:**
- `PESQ` — значение, полученное из `compute_pesq_approx()`
- `1.3669`, `0.7197`, `3.87` — эмпирические константы из стандарта ITU-T P.862.1 (Mapping function for transforming PESQ scores to MOS-LQO)
- Формула представляет собой логистическую кривую (сигмоиду) с диапазоном [1, 4.87]
- Шкала оценки:
  - 5 — Отлично (неотличимо от оригинала)
  - 4 — Хорошо (заметные, но не мешающие искажения)
  - 3 — Удовлетворительно (заметные искажения)
  - 2 — Плохо (существенные искажения)
  - 1 — Неприемлемо

## 11. Агрегированный Score (интегральная оценка)

**Функция:** `compute_metrics_batch(...)` — итоговый балл качества метода

**Описание.** Взвешенная сумма нормализованных метрик. Все метрики сначала приводятся к единой шкале [0, 1] с помощью min-max нормализации, затем складываются с весами. Чем выше Score, тем лучше метод обработки.

### 11.1. Min-Max нормализация

Для метрик **«ниже — лучше»** (LSD, Spectral Convergence, RMSE, Centroid Diff, Time):
```
norm = (max - value) / (max - min + eps)
```
(Инверсия: наилучшее (минимальное) значение получает норму 1.0)

Для метрик **«выше — лучше»** (SNR, SI-SDR, Cosine, STOI, PESQ, MOS):
```
norm = (value - min) / (max - min + eps)
```
(Наилучшее (максимальное) значение получает норму 1.0)

Где:
- `min`, `max` — минимальное и максимальное значения метрики среди всех сравниваемых методов для данного файла
- `eps = 1e-12` — защита от деления на ноль

### 11.2. Взвешенная сумма

```
Score = w_lsd     * LSD_norm
      + w_snr     * SNR_norm
      + w_rmse    * RMSE_norm
      + w_si_sdr  * SI-SDR_norm
      + w_sc      * SC_norm
      + w_cd      * Centroid_diff_norm
      + w_cos     * Cosine_norm
      + w_time    * Time_norm
      + w_stoi    * STOI_norm
      + w_pesq    * PESQ_norm
      + w_mos     * MOS_norm
```

**Веса (из `MetricsConfig`):**

| Метрика | Вес | Обозначение | Тип |
|---|---|---|---|
| LSD | 0.15 | `w_lsd` | ниже лучше |
| SNR | 0.15 | `w_snr` | выше лучше |
| RMSE | 0.10 | `w_rmse` | ниже лучше |
| SI-SDR | 0.10 | `w_si_sdr` | выше лучше |
| Spectral Convergence | 0.10 | `w_sc` | ниже лучше |
| Centroid Diff | 0.05 | `w_cd` | ниже лучше |
| Cosine Similarity | 0.05 | `w_cos` | выше лучше |
| Time | 0.05 | `w_time` | ниже лучше |
| STOI | 0.10 | `w_stoi` | выше лучше |
| PESQ | 0.10 | `w_pesq` | выше лучше |
| MOS | 0.05 | `w_mos` | выше лучше |
| **Итого** | **1.00** | | |
