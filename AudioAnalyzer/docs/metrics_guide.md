## 1. Зачем нужны метрики

### 1.1. Проблема сравнения аудио

При обработке аудиосигналов возникает вопрос: **как объективно оценить качество результата?** Человеческое восприятие звука субъективно — то, что кажется качественным одному человеку, может не понравиться другому.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        ПРОБЛЕМА СУБЪЕКТИВНОСТИ                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Субъективные оценки:                                                       │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐                     │
│  │ "Звучит     │    │ "Нормально" │    │ "Какое-то   │                     │
│  │  отлично!"  │    │             │    │  дребезжание"│                     │
│  └─────────────┘    └─────────────┘    └─────────────┘                     │
│        ↓                   ↓                   ↓                          │
│  Один и тот же файл, разные мнения                                          │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

Для научного сравнения нужны ОБЪЕКТИВНЫЕ метрики — численные критерии,
которые можно измерить и сравнить.
```

### 1.2. Почему недостаточно одной метрики

Каждая метрика оценивает только один аспект качества:

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                     МНОГОГРАННОСТЬ КАЧЕСТВА                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│                     Качество аудио                                          │
│                          │                                                  │
│        ┌─────────────────┼─────────────────┐                               │
│        │                 │                 │                               │
│    Временная         Спектральная      Восприятие                          │
│    область           область           (перцептивное)                      │
│        │                 │                 │                               │
│   ┌────┴────┐       ┌────┴────┐       ┌────┴────┐                         │
│   SNR      RMSE     LSD      CosSim   (требует                                │
│   SI-SDR            Conv     CentrΔ   психоакуст.                           │
│                                        модели)                              │
│                                                                             │
│  Ни одна метрика не покрывает все аспекты → нужен набор метрик             │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 2. Метрики временной области

Метрики временной области сравнивают отсчёты сигналов напрямую, без преобразования в спектр.

### 2.1. SNR (Signal-to-Noise Ratio)

**Что измеряет:** Отношение мощности полезного сигнала к мощности шума (ошибки).

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              SNR (дБ)                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Формула:                                                                   │
│                                                                             │
│                    P_signal                                                 │
│      SNR = 10·log₁₀(────────)                                               │
│                    P_noise                                                  │
│                                                                             │
│  где:                                                                       │
│      P_signal = mean(ref²)           — мощность сигнала                    │
│      P_noise  = mean((ref - test)²)  — мощность ошибки (шума)              │
│                                                                             │
│  Интерпретация:                                                             │
│  ┌─────────────────────────────────────────────────────────────────────────┐│
│  │ SNR > 40 дБ  ████████████████████  Отличное качество                  ││
│  │ SNR 30-40 дБ ████████████████░░░░  Хорошее качество                   ││
│  │ SNR 20-30 дБ ████████░░░░░░░░░░░░  Заметные искажения                 ││
│  │ SNR < 20 дБ  ████░░░░░░░░░░░░░░░░  Плохое качество                   ││
│  └─────────────────────────────────────────────────────────────────────────┘│
│                                                                             │
│  ↑ выше = лучше                                                             │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**Пример расчёта:**

```python
# Исходный сигнал
ref = np.array([0.5, 0.7, 0.3, -0.2, -0.5])

# Обработанный сигнал (с искажениями)
test = np.array([0.48, 0.72, 0.28, -0.18, -0.52])

# Расчёт SNR
P_signal = np.mean(ref ** 2)      # = 0.264
P_noise = np.mean((ref - test) ** 2)  # = 0.0008
SNR = 10 * np.log10(P_signal / P_noise)  # = 25.2 дБ
```

### 2.2. RMSE (Root Mean Square Error)

**Что измеряет:** Среднеквадратичную ошибку между сигналами.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              RMSE                                           │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Формула:                                                                   │
│                                                                             │
│      RMSE = √(mean((ref - test)²))                                          │
│                                                                             │
│  Физический смысл:                                                          │
│  Среднее отклонение амплитуды между сигналами.                             │
│ Измеряется в тех же единицах, что и сигнал (для PCM: 0..1).                │
│                                                                             │
│  Интерпретация:                                                             │
│  ┌─────────────────────────────────────────────────────────────────────────┐│
│  │ RMSE < 0.01   ████████████████████  Минимальные искажения             ││
│  │ RMSE 0.01-0.05 ████████████░░░░░░░░  Заметные искажения               ││
│  │ RMSE > 0.05   ████░░░░░░░░░░░░░░░░  Сильные искажения                 ││
│  └─────────────────────────────────────────────────────────────────────────┘│
│                                                                             │
│  ↓ ниже = лучше                                                             │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.3. SI-SDR (Scale-Invariant Signal-to-Distortion Ratio)

**Что измеряет:** Отношение сигнал/искажение, инвариантное к масштабированию.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                            SI-SDR (дБ)                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Зачем нужен:                                                               │
│  SNR чувствителен к масштабу сигнала. Если test = 2·ref, SNR упадёт,       │
│  хотя сигнал просто громче! SI-SDR решает эту проблему.                    │
│                                                                             │
│  Формула:                                                                   │
│                                                                             │
│      α = <ref, test> / <ref, ref>     — оптимальный масштаб               │
│      s_target = α · ref                — целевой компонент                │
│      e_noise = test - s_target         — шум                              │
│                                                                             │
│               ||s_target||²                                                 │
│      SI-SDR = 10·log₁₀(───────────)                                         │
│               ||e_noise||²                                                  │
│                                                                             │
│  Интерпретация:                                                             │
│  ┌─────────────────────────────────────────────────────────────────────────┐│
│  │ SI-SDR > 20 дБ  ████████████████████  Отличное качество               ││
│  │ SI-SDR 10-20 дБ ████████████░░░░░░░░  Хорошее качество                ││
│  │ SI-SDR < 10 дБ  ████░░░░░░░░░░░░░░░░  Плохое качество                 ││
│  └─────────────────────────────────────────────────────────────────────────┘│
│                                                                             │
│  ↑ выше = лучше                                                             │
│  Не зависит от громкости сигнала!                                          │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 3. Метрики спектральной области

Метрики спектральной области сравнивают частотные характеристики сигналов, полученные через FFT.

### 3.1. LSD (Log-Spectral Distance)

**Что измеряет:** Расстояние между логарифмическими спектрами.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           LSD (дБ)                                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Формула:                                                                   │
│                                                                             │
│      S_ref = 20·log₁₀(|R| + ε)     — лог. спектр референса                │
│      S_test = 20·log₁₀(|T| + ε)    — лог. спектр теста                    │
│                                                                             │
│                √(mean((S_ref - S_test)²))                                   │
│      LSD = ─────────────────────────                                        │
│                              1                                              │
│                                                                             │
│  Усредняется по всем окнам анализа.                                        │
│                                                                             │
│  Интерпретация:                                                             │
│  ┌─────────────────────────────────────────────────────────────────────────┐│
│  │ LSD < 1 дБ   ████████████████████  Почти идентичные спектры           ││
│  │ LSD 1-3 дБ   ████████████░░░░░░░░  Заметные различия                  ││
│  │ LSD > 3 дБ   ████░░░░░░░░░░░░░░░░  Сильные различия                   ││
│  └─────────────────────────────────────────────────────────────────────────┘│
│                                                                             │
│  ↓ ниже = лучше                                                             │
│                                                                             │
│  Особенность: логарифмическая шкала соответствует восприятию человека      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3.2. Spectral Convergence

**Что измеряет:** Нормированную ошибку амплитудных спектров.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      Spectral Convergence                                   │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Формула:                                                                   │
│                                                                             │
│              |||R| - |T|||₂                                                 │
│      SC = ─────────────────                                                 │
│                  ||R||₂                                                     │
│                                                                             │
│  где |R|, |T| — амплитудные спектры                                        │
│       ||·||₂ — L2-норма (корень из суммы квадратов)                        │
│                                                                             │
│  Интерпретация:                                                             │
│  ┌─────────────────────────────────────────────────────────────────────────┐│
│  │ SC < 0.05    ████████████████████  Хорошее совпадение спектров        ││
│  │ SC 0.05-0.15 ████████████░░░░░░░░  Заметные различия                  ││
│  │ SC > 0.15    ████░░░░░░░░░░░░░░░░  Сильные различия                   ││
│  └─────────────────────────────────────────────────────────────────────────┘│
│                                                                             │
│  ↓ ниже = лучше                                                             │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3.3. Spectral Centroid Difference

**Что измеряет:** Разницу «центров тяжести» спектров.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      Centroid Δ (Гц)                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Понятие центроида:                                                         │
│  Центроид спектра — это «средняя частота» спектра, точка равновесия        │
│  амплитудного спектра.                                                      │
│                                                                             │
│  Формула:                                                                   │
│                                                                             │
│               Σ(k · |X_k|)                                                  │
│      centroid = ───────────── × (sample_rate / N)                           │
│                  Σ|X_k|                                                     │
│                                                                             │
│  где k — индекс частотного бина                                             │
│                                                                             │
│  Centroid Δ = mean(|centroid_ref - centroid_test|)                         │
│                                                                             │
│  Интерпретация:                                                             │
│  ┌─────────────────────────────────────────────────────────────────────────┐│
│  │ Δ < 100 Гц   ████████████████████  Спектры похожи по «яркости»        ││
│  │ Δ 100-500 Гц ████████████░░░░░░░░  Заметный сдвиг                      ││
│  │ Δ > 500 Гц   ████░░░░░░░░░░░░░░░░  Сильное различие                   ││
│  └─────────────────────────────────────────────────────────────────────────┘│
│                                                                             │
│  ↓ ниже = лучше                                                             │
│                                                                             │
│  Физический смысл:                                                         │
│  • Низкий центроид — басовый, «глухой» звук                                │
│  • Высокий центроид — яркий, «звонкий» звук                                │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3.4. Cosine Similarity

**Что измеряет:** Косинус угла между спектральными векторами.

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      Cosine Similarity                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Формула:                                                                   │
│                                                                             │
│              <R, T>                                                         │
│      CosSim = ────────────                                                  │
│              ||R|| · ||T||                                                  │
│                                                                             │
│  где <R, T> — скалярное произведение                                       │
│       ||R|| — норма вектора                                                │
│                                                                             │
│  Геометрическая интерпретация:                                             │
│                                                                             │
│      R ●                    R ●                                             │
│      │╲                     │  ╲                                            │
│      │ ╲ T                  │    ╲ T                                        │
│      └───►                  └──────►                                        │
│   CosSim ≈ 0.9             CosSim ≈ 0.5                                     │
│   (малый угол)             (большой угол)                                   │
│                                                                             │
│  Интерпретация:                                                             │
│  ┌─────────────────────────────────────────────────────────────────────────┐│
│  │ CosSim > 0.95  ████████████████████  Высокая схожесть                 ││
│  │ CosSim 0.85-0.95 ████████████░░░░░░  Хорошая схожесть                  ││
│  │ CosSim < 0.85  ████░░░░░░░░░░░░░░░░  Низкая схожесть                  ││
│  └─────────────────────────────────────────────────────────────────────────┘│
│                                                                             │
│  ↑ выше = лучше (диапазон 0..1)                                            │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 4. Агрегированный балл

### 4.1. Зачем нужен общий балл

При сравнении 7 методов по 8 метрикам получается 56 чисел. Общий балл позволяет:
- Ранжировать методы одним числом
- Учитывать разные аспекты качества
- Сравнивать результаты обработки

### 4.2. Методика расчёта

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    РАСЧЁТ АГРЕГИРОВАННОГО БАЛЛА                            │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ШАГ 1: Min-Max нормировка                                                  │
│  ─────────────────────────                                                  │
│  Все метрики приводятся к диапазону [0, 1] по всем сравниваемым методам.   │
│                                                                             │
│  Для метрик «ниже-лучше» (LSD, SC, RMSE, CentroidΔ, time):                 │
│                                                                             │
│                    max - value                                               │
│      norm = ───────────────────                                             │
│                   max - min                                                 │
│                                                                             │
│  Для метрик «выше-лучше» (SNR, SI-SDR, CosSim):                            │
│                                                                             │
│                    value - min                                               │
│      norm = ───────────────────                                             │
│                   max - min                                                 │
│                                                                             │
│  Результат: все нормированные значения → «выше = лучше»                    │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

ШАГ 2: Взвешенная сумма
─────────────────────────────────────────────────────────────────────────────────

┌─────────────────────────────────────────────────────────────────────────────┐
│  Метрика            Вес     Обоснование                                     │
│  ────────────────────────────────────────────────────────────────────────── │
│  LSD (дБ)           20%     Главный показатель спектрального качества      │
│  SNR (дБ)           20%     Классическая метрика временной области         │
│  RMSE               15%     Точность во временной области                  │
│  SI-SDR (дБ)        15%     Устойчивость к масштабу                        │
│  Spectral Conv.     15%     Спектральная точность                          │
│  Centroid Δ (Гц)    5%      Характер звучания                              │
│  Cosine Sim.        5%      Общая схожесть спектров                        │
│  Time (с)           5%      Скорость обработки                             │
│  ────────────────────────────────────────────────────────────────────────── │
│  Итого:             100%                                                    │
│                                                                             │
│  Формула:                                                                   │
│  score = 0.20·LSD_n + 0.20·SNR_n + 0.15·RMSE_n + 0.15·SI_SDR_n +          │
│          0.15·SC_n + 0.05·Centroid_n + 0.05·CosSim_n + 0.05·Time_n         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────────────────────┤

│                                                                             │

```

## Заключение

### Ключевые принципы использования метрик:

1. **Не ориентируйтесь на одну метрику** — каждая оценивает только один аспект качества
2. **Учитывайте тип аудио** — разные методы лучше для разных жанров
3. **Используйте Score для ранжирования** — но анализируйте отдельные метрики
4. **Сравнивайте в рамках одного набора** — Score относителен

### Рекомендации по выбору метода:

| Тип аудио | Рекомендуемый метод | Обоснование |
|-----------|---------------------|-------------|
| Классика | DCT, FFT | Высокое качество, широкий спектр |
| Рок/Метал | FWHT, FFT | Баланс скорости и качества |
| Электроника | DCT, FFT | Важны высокие частоты |
| Речь | Huffman (bits=8+) | Эффективно для узкополосного сигнала |
| Ударные | DWT | Локализация переходов |
